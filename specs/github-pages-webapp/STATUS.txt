DEBUGGING TIPS (if local testing fails)
================================================================================

ISSUE: JavaScript errors in console
- Open browser console (F12) and read the error message
- Check that all function names in index.html match those in script.js
- Verify modal element IDs (#modal, #videoContainer, #scenarioInfo, etc.)
- Ensure script.js is loaded (check Network tab in dev tools)

ISSUE: Videos don't play
- Check that video files exist in docs/ directory (they do!)
- Verify file paths in SCENARIOS object match actual filenames
- Test video playback directly: open docs/01_normal_operation.mp4 in browser
- Check browser console for video loading errors
- If MP4 fails, GIF fallback should activate automatically

ISSUE: Modal doesn't open
- Check that onclick handlers in index.html call correct functions
- Verify viewScenario() and showDetails() functions exist in script.js
- Check that scenario IDs in HTML match keys in SCENARIOS object
- Look for JavaScript errors preventing function execution

ISSUE: Modal doesn't close
- Verify closeModal() function exists and is called by:
  - Close button onclick handler
  - Backdrop click event listener
  - Escape key event listener
- Check that modal element has ID "modal"
- Ensure event listeners are registered (check script.js bottom section)
